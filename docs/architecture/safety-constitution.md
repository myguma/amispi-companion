# Safety Constitution — AmitySpirit Companion

## 設計原則 (変更禁止)

```
Observe locally.    ローカルで見る。
Interpret gently.   やさしく解釈する。
Suggest softly.     控えめに提案する。
Never mutate.       勝手に変更しない。
```

この原則はコードレビューの基準であり、機能追加の判断基準である。

---

## 初期MVPで禁止すること

以下は**実装してはならない**。feature flagでも原則禁止。

| 禁止項目 | 理由 |
|---------|------|
| ファイルの削除・移動・リネーム・上書き | Mutate |
| shell コマンド実行 | Mutate + 危険 |
| ブラウザ自動操作 | Mutate |
| メール・SNS 送信 | Mutate + 外部送信 |
| クリップボードの常時監視 | プライバシー侵害 |
| 画面常時 OCR | プライバシー侵害 |
| キーログ・入力内容の取得 | プライバシー侵害 |
| ユーザーに見えないクラウド送信 | 信頼破壊 |
| Windows ハックによる他アプリ制御 | 危険 |
| 長期履歴の自動保存 (タイトル・ファイル名) | プライバシー侵害 |

## 将来の機能追加条件

操作系機能 (Mutate) を追加する場合は:
1. 明示的ユーザー承認 (毎回)
2. dry-run プレビュー表示
3. Undo 機能
4. 操作ログの永続保存
5. 設定からいつでも無効化できること

この条件を満たさない操作機能はマージしない。

## PrivacyGate の役割

- LLM に「秘密を守って」と頼るのではなく、**そもそも渡さない**
- cloud_llm への送信は cloudAllowed=true でもウィンドウタイトル・ファイル名は禁止
- clipboard・screenContent は宛先を問わず常に deny
- UI デバッグログにもタイトル・ファイル名を出さない

## PermissionBoundary の役割

- React (フロントエンド) は UI 表示だけを担当する
- 危険な判断・フィルタリングは Rust バックエンドで行う
- フロントエンドに生の観測データを渡す前に Rust 側でフィルタリングする

## cloud off by default

- cloudAllowed のデフォルトは **false**
- ユーザーが明示的に ON にした場合のみ外部送信が許可される
- 送信前に「何を送るか」をプレビュー表示する (将来実装)
- API キーはこのファイルやコードに埋め込まない

## no invisible memory

- ウィンドウタイトル履歴を保存しない
- ファイル名履歴を保存しない
- ファイル内容を保存しない
- クリップボード内容を保存しない
- 保存する情報: 設定・権限変更履歴・集計統計 (将来)
